Informe Técnico - Análisis Dinámico de la Muestra de Malware
1. Introducción al Análisis Dinámico
El análisis dinámico consiste en ejecutar la muestra de malware en un entorno controlado (como una máquina virtual) para observar su comportamiento en tiempo real. A diferencia del análisis estático, este enfoque permite identificar acciones maliciosas como la creación de archivos, conexiones de red, modificaciones en el registro y más.

El objetivo de este análisis es complementar la información obtenida en el análisis estático y entender cómo el malware interactúa con el sistema y la red. Además, se utilizaron herramientas especializadas para monitorear el tráfico de red, los procesos y los cambios en el sistema.

2. Metodología del Análisis Dinámico
El análisis dinámico se realizó siguiendo estos pasos:

Entorno Controlado:
Se utilizó una máquina virtual con Windows 10 Professional (aislada de la red principal) para ejecutar la muestra. La máquina virtual estaba configurada con herramientas de monitoreo y sin conexión a internet, excepto cuando era necesario observar el tráfico de red.

Herramientas de Monitoreo:
Se emplearon las siguientes herramientas para observar el comportamiento del malware:

Process Monitor (ProcMon): Para monitorear la creación de procesos, archivos y modificaciones en el registro.

Wireshark: Para capturar y analizar el tráfico de red generado por el malware.

ProcDOT: Para visualizar de manera gráfica la actividad del malware en el sistema.

Regshot: Para comparar los cambios en el registro de Windows antes y después de la ejecución del malware.

Ejecución de la Muestra:
La muestra redteam-05.png se ejecutó en la máquina virtual mientras se monitoreaban sus acciones. Se prestó especial atención a:

Creación de archivos y procesos.

Conexiones de red y dominios contactados.

Modificaciones en el registro de Windows.

Contraste con Plataformas de Análisis:
Los resultados obtenidos se compararon con la información proporcionada por plataformas como Any.Run, VirusTotal y Hybrid Analysis para enriquecer el análisis y validar los hallazgos.

3. Resultados del Análisis Dinámico
3.1. Comportamiento Observado
Durante la ejecución de la muestra, se observaron las siguientes actividades maliciosas, consistentes con los resultados de Any.Run:

Actividades Maliciosas (MALICIOUS):

Modificación de Archivos:
El proceso Vindows.exe (PID: 6448) modificó archivos en la carpeta de extensiones de Chrome, lo que sugiere un intento de robo de datos personales.

Robo de Datos:
El mismo proceso realizó acciones consistentes con el robo de información personal.

Comportamiento de Ransomware:
Se observó que el malware renombró archivos, lo que es típico de un ataque de ransomware.

Actividades Sospechosas (SUSPICIOUS):

Creación de Procesos Legítimos:
El proceso WinRAR.exe (PID: 1744) creó archivos con nombres similares a los de archivos del sistema, lo que podría ser un intento de evadir detección.

Uso de PasteBin:
El proceso Vindows.exe (PID: 6448) solicitó información desde PasteBin, lo que sugiere exfiltración de datos.

Ejecución de Aplicaciones que Fallan:
El mismo proceso intentó ejecutar aplicaciones que terminaron fallando, lo que podría indicar errores en el código malicioso.

Información Adicional (INFO):

Uso de Archivos Descargados:
El proceso WinRAR.exe (PID: 1744) utilizó archivos descargados, lo que podría indicar la descarga de componentes adicionales.

Ejecución Manual por el Usuario:
Varios procesos, como mspaint.exe y decryptor.exe, fueron ejecutados manualmente, lo que sugiere interacción directa con el malware.

Lectura de Información del Sistema:
El proceso Vindows.exe (PID: 6448) leyó el GUID de la máquina, el nombre del equipo y verificó los idiomas soportados, lo que podría ser utilizado para personalizar el ataque.

Creación de Archivos Cifrados:
El malware creó archivos cifrados y scripts VBS (codificados con Microsoft Script Encoder) en directorios temporales.

Deshabilitación de Registros:
El proceso Vindows.exe (PID: 6448) deshabilitó los registros de trazas, lo que dificulta la detección y análisis del malware.

3.2. Tráfico de Red
Utilizando Wireshark, se capturó el tráfico de red generado por el malware. Se identificaron las siguientes actividades:

Conexiones a dominios sospechosos relacionados con el malware.

Intentos de exfiltrar datos sensibles a través de Pastebin.

Uso de protocolos no cifrados para comunicarse con servidores de comando y control (C2).

3.3. Comparación con Plataformas de Análisis
Los resultados obtenidos coinciden con los reportes de Any.Run, donde se identificó actividad maliciosa relacionada con ransomware y stealer. Además, se validaron los hallazgos con VirusTotal y Hybrid Analysis, lo que confirma el comportamiento observado en el entorno controlado.

4. Herramientas Utilizadas
Process Monitor (ProcMon): Para monitorear procesos, archivos y registros.

Wireshark: Para analizar el tráfico de red.

ProcDOT: Para visualizar la actividad del malware.

Regshot: Para comparar cambios en el registro.

Any.Run: Para contrastar el comportamiento observado.

5. Conclusiones
El análisis dinámico de la muestra redteam-05.png permitió observar su comportamiento en tiempo real, identificando actividades maliciosas como la creación de procesos, conexiones de red y modificaciones en el registro. Estos hallazgos complementan la información obtenida en el análisis estático y confirman que la muestra es un ransomware con capacidades de stealer.

La combinación de herramientas de monitoreo y el contraste con plataformas de análisis como Any.Run y VirusTotal enriquecieron el informe, proporcionando una visión completa del comportamiento del malware.
